# H-drive runtime cache pipeline

## Goal

Move long-run pipeline transient files and framework caches away from `C:` and onto `H:` so training can keep running with low system-drive free space.

## Files touched

- `scripts/pipeline.py`
- `scripts/pipeline_register.ps1`
- `tests/test_pipeline_orchestrator.py`

## Key decisions

- Use `H:/elt_data/cache` as the canonical runtime cache root.
- Force `TMP`, `TEMP`, `TMPDIR`, `UV_CACHE_DIR`, `UV_PYTHON_INSTALL_DIR`, `UV_TOOL_DIR`, `PIP_CACHE_DIR`, `HF_HOME`, `HF_HUB_CACHE`, `HF_DATASETS_CACHE`, `TRANSFORMERS_CACHE`, `TORCH_HOME`, `XDG_CACHE_HOME`, `TRITON_CACHE_DIR`, `TORCHINDUCTOR_CACHE_DIR`, `CUDA_CACHE_PATH`, `NUMBA_CACHE_DIR`, `MPLCONFIGDIR`, and `PYTHONPYCACHEPREFIX` to `H:/elt_data/cache/*` for pipeline child processes.
- Add the same environment setup to the Windows Task Scheduler launcher generated by `scripts/pipeline_register.ps1`.
- Add `-WriteLauncherOnly` so the launcher can be refreshed without unregistering or re-registering the currently running scheduled task.

## Tests

- Added pipeline orchestrator coverage for H-drive subprocess environment creation.
- Added coverage that `vsdev_command(...)` propagates H-drive cache variables into its `cmd.exe` wrapper.

## Next session notes

- The already-running Python pipeline process keeps its inherited environment until restarted.
- Use `powershell -ExecutionPolicy Bypass -File scripts/pipeline_register.ps1 -StartLongTrain -Profile posttrain-grpo -WriteLauncherOnly` to refresh only `H:/elt_data/pipeline_logs/pipeline_launcher.ps1`.
- If `C:` remains critically low during the current run, restart the scheduled task after a fresh rolling checkpoint so the pipeline resumes from `last.pt` with the H-drive cache environment.
